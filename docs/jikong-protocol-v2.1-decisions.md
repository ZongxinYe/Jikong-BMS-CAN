# 极空 BMS CAN V2.1 解析约定

本项目以 `极空BMS-Can协议V2.1.pdf` 的字段表和示例帧为依据。协议中少量表格、示例存在冲突，Phase 1 按以下规则固化，后续用真机报文继续校验。

## 已固化规则

- 除 `BMSChgINFO (0x1806E5F4)` 外，多字节数值默认按小端解析。
- `BMSChgINFO` 的充电电压和充电电流按大端解析。
- DBC 使用默认设备地址 `F4` 的帧 ID；解码前从实际 ID 中减去 APP 配置的设备地址。
- 当前设备地址限制为 `0..11`，因为标准帧 `0x07F4` 再增加更大地址会超过 11 位 CAN ID 上限 `0x7FF`。
- `ALL_TEMP` 中掩码未启用或原始值为 `0xFF` 的温度，在补充解析层输出为 `None`。
- `CellVol` 七个扩展帧按每帧四节组装，形成第 1 至第 25 节电压；填充值 `0x0000` 不作为有效单体。

## PDF 冲突处理

| 位置 | PDF 冲突 | 当前实现 |
| --- | --- | --- |
| `CELL_VOLT`、`CELL_TEMP` 位置字段 | 表格写偏移量 1，但示例把原始 `05`、`06` 直接解释为第 5、6 节 | DBC 使用偏移量 0，与示例一致 |
| `BATT_ST1.SOC` | 表格范围写 `0~1`，示例 `0x33` 表示 51% | DBC 范围使用 `0~100%` |
| `BATT_ST2.CycleCount` | 总体约定和起始位表暗示小端，示例 `00 64` 却按大端解释为 100 | DBC 按示例使用大端；补充层在大端结果超过 1000 且小端结果合理时自动回退到小端 |
| `BMSERR_INFO` bit 8 之后 | 表格从 bit 8 连续排列，但示例 `02 30 01` 无法得到文中故障 | bit 8 作为保留位，后续字段使用 bit 9..18，使示例得到 MOS 过温、单体欠压、总压欠压、放电温度过高 |
| `BMS_INFO` 示例排版 | 示例把 `C8 00`、`00 00 28`、`0A 64` 分组，与字段位宽不一致 | 按字段表解析为 `C8 00 00 00` 运行时间、`28 0A` 加热电流、`64` SOH |

## 真机验证重点

连接 CANalyst-II 后优先保存以下原始帧，用来确认协议勘误选择：

- `0x18F128F4` 的第 6、7 字节，确认循环次数实际字节序。
- `0x18F328F4` 的故障位，确认 bit 8 是否确实保留。
- 非零设备地址下的标准帧和扩展帧，确认两类 ID 都采用直接加地址规则。
