## 只读接收验收

设备接入后先执行只枚举：

```powershell
python -B tools\canalyst2_check.py
```

确认设备信息正确后，使用只接收方式打开 CAN1 五秒：

```powershell
python -B tools\canalyst2_check.py --channel 0 --bitrate 250000 --receive-seconds 5
```

通过条件：

- 不返回 `0xFFFFFFFF` 或适配器错误。
- 能收到周期帧，帧 ID、标准/扩展标志和 DLC 合理。
- GUI 总压、电流、SOC、单体和波形持续更新。
- 接收队列深度不会持续增长，解析失败数量稳定。

## 长稳验收

### 30 分钟接收

- 使用只听模式，不开启控制。
- 每 5 分钟记录内存、帧率、队列深度和丢帧数。
- 通过条件：GUI 操作正常，队列回落，无持续内存异常增长。

### 2 小时记录

- 使用正常模式但保持控制锁定。
- 开启 SQLite 记录和三条实时波形。
- 结束后正常停止记录，运行 `tools\export_recording.py` 导出。
- 通过条件：数据库完整、原始帧和信号行数合理、CSV 可读取、停止过程无写入错误。

## 异常验收

1. 接收中拔掉 USB：程序不崩溃，显示适配器错误，记录会话可正常结束。
2. 无 CAN 报文：出现接收超时事件，UI 仍可操作。
3. 错误设备地址：保留原始帧但解析计数增加，修正地址后恢复。
4. 数据库文件占用或目录不可写：记录明确失败，不影响只读监测。
5. 审计目录不可写：控制命令在调用适配器前被拒绝。

## 控制验收门槛

真实 `Ctrl_INFO` 发送必须单独审批，并满足：台架无人员接触高压、急停可用、充放电回路状态已知、目标为默认地址 0、近期已收到目标 BMS 报文、独立审计文件可写。首次只选择一个 Mask 位并发送一帧，随后通过 `BMS_SWITCH_STATUS` 和外部测量共同确认结果。
